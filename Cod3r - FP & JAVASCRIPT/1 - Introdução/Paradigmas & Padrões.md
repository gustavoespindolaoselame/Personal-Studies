# Paradigmas & Padrões - Modelo ou padrão a seguir
------------------------------------------------------------------------------------------------------------

    Paradigmas são a estrutura que se usa para definir o software.
        Por exemplo analisar o mundo como objetos e suas relações, com os objetos tendo seus atributos e responsabilidades etc.
        No mundo funcional, você analisa o mundo por meio de uma série de funções.
        Todos paradigmas tem diferentes benefícios e problemas.
        Você pode ter uma soma de paradigmas em seu projeto.
        Paradigmas são mais difíceis de aprender novos do que linguagens.

    Padrões tem como objetivo resolver um problema comum.
        Por exemplo como organizar sua aplicação, não querer misturar o acesso ao banco de dados à interface, esse seria um padrão layer.
        Padrões pertencem à paradigmas.

    Ordem de desenvolvimento de paradigmags;
        * Funcional - 1957 
            Iníciou com LISP.
            Trabalha com dados constantes consumindo mais memória.
            Melhor para código em paralelo

        * Procedural - 1968
            Trabalha com dados mutáveis.

        * Orientada à objetos - 1980/90
            Trabalha com dados mutáveis.
